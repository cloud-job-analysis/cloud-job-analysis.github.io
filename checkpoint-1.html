Checkpoint 1

Current Status

We analyzed the working of Mesos. The master node determines how many resources to offer to each framework. In here,

  1. The agents will inform the mesos master of the resources it contains
  2. The Mesos Master will send a resource offer to framework 1.
  3. Framework will decide which resources to utilize using tasks
  4. Master sends tasks to agents.

Since we are working with individual jobs instead of frameworks, steps 2 + 3 become redundant. Instead, the Mesos Master itself should act as the framework scheduler for the individual jobs, denying resource offers that aren’t relevant. In order to satisfy the constraints of a framework, the framework themselves would usually have the ability to deny resource offers. Instead, we’d have to have the master node take on this task.A model of the Mesos master has been implemented.

We looked into simulating and integrating jobs calls from different frameworks like Spark, MapReduce and nginx. This helped decide features for the jobs. This serves as a sanity check as to whether our architecture can support real-life processes.

We have decided various properties for each job that will help our algorithm make good predictions. These are in tune with attributes related to Apache Spark and nginx processes. Presently, we categorise jobs into 4 basic types - web-based jobs, machine learning jobs, database jobs and indexing jobs. Each of these have distinct characteristics which affect the performance of scheduling algorithms. For use in the algorithm, values of these characteristics are generated by sampling from a Gaussian distribution.

We have looked at various scheduling algorithms to compare our approach to. The master node in Mesos uses the Dominant Resource Fairness (DRF) algorithm. So that is the first baseline we implemented. Similarly, we have also implemented Shortest Job First (SJF).

We have completed designing the states and flow for the master node and agents (hosts). This depicts the various stages of our system and gives a high-level overview of its basic structure.
