About

In modern large, parallel systems such as single-tenant data centers and scientific clusters there are typically many thousands of active or to-be-run jobs at any given time. Each job has its own requirements for time and space in order to run properly. Due to resource constraints it is not possible to run each job simultaneously and so scheduling algorithms are necessary in order to assign an ordering and allocate resources to the jobs submitted to the system.

Scheduling algorithms generally try to optimize one or more criteria such as fairness or priority. In modern systems, the biggest concern in selecting an algorithm is utilization, since even small inefficiencies can cost millions of dollars. Algorithms like Distributed Shortest Remaining Time First (D-SRTF) and earliest deadline first (EDF) address this issue by optimizing throughput. Unfortunately, these algorithms are predicated on the ability to reliably predict the run time of jobs, which is typically only possible under strong assumptions about the system and job type.

In this project we aim to use machine learning (ML) modeling to obtain better predictions of job run-times similar to the efforts in helmy2015machine. This will enable the use of run-time-based scheduling algorithms in more general traffic scenarios.
